# 数据采集指南 v4

## 目标
采集 beam 节奏型，训练节奏型分类器

---

## 核心原则

**只画符干 + beam，不画符头！**

```
关键信息:           噪声 (不管):
- 符干数量            - 符头 (冗余)
- beam数量           - 符干长度 (归一化)
- 连接关系            - beam斜率 (不影响节奏)
```

---

## 一、节奏型采集

### 1. beam_8_8 - 采集 40 个

**两个八分音符 (2根符干 + 1条beam)**

```
│───│
│   │
```

随便画，直的斜的都行。

---

### 2. beam_16_16 - 采集 40 个

**两个十六分音符 (2根符干 + 2条beam)**

```
│═══│
│═══│
│   │
```

两条beam要能区分开。

---

### 3. beam_16_16_8 - 采集 35 个

**前十六后八 (3根符干，前两根双beam)**

```
│══│──│
│══│  │
│  │  │
```

- 主beam连全部
- 次beam只连前两根

---

### 4. beam_8_16_16 - 采集 35 个

**前八后十六 (3根符干，后两根双beam)**

```
│──│══│
│  │══│
│  │  │
```

- 主beam连全部
- 次beam只连后两根

---

### 5. beam_16x4 - 采集 30 个

**四个十六分音符 (4根符干 + 2条beam)**

```
│══│══│══│
│══│══│══│
│  │  │  │
```

---

### 6. beam_triplet - 采集 25 个

**三连音 (3根符干 + 1条beam)**

```
  3
│─│─│
│ │ │
```

可以画"3"标记，也可以不画。

---

## 二、采集汇总

| 类别 | 标签名 | 数量 |
|------|--------|------|
| 两个八分 | `beam_8_8` | 40 |
| 两个十六分 | `beam_16_16` | 40 |
| 前十六后八 | `beam_16_16_8` | 35 |
| 前八后十六 | `beam_8_16_16` | 35 |
| 四个十六分 | `beam_16x4` | 30 |
| 三连音 | `beam_triplet` | 25 |
| **总计** | | **205** |

---

## 三、画法要点

### 要
- 符干数量正确
- beam数量正确
- beam连到符干

### 不管
- 斜率随意
- 符干长短随意
- 不用画符头

### 笔画顺序 (建议)
1. 先画符干 (从左到右)
2. 再画beam (横着连)

## 四、示例

```
正确 ✓                    也正确 ✓
```

## 六、识别逻辑

```
输入: 用户画的符干+beam
      ↓
模型识别: beam_8_8 / beam_16_16 / ...
      ↓
DaSegno: 知道是几个什么时值的音
      ↓
渲染: 自动加符头、算beam斜率
```

用户只需要画"骨架"，DaSegno负责美化。
